# 6. Generate the new Connector Source

You can create a new connector source in two ways:

## A. Generate from csp-api UI

Use this option when you want the SDK to generate a starter connector module from the UI:
1. Start `csp-api` from the repository root:

```
mvn clean install -pl csp-api -am
java -jar csp-api/target/csp-api-*.jar
```

2. Open the UI at `http://localhost:8800/` and click **Create New Connector**.

![](../assets/Home.png)

In the **Create New Connector** form:
- **Connector Name / Data Source**: Connector name (for example, `MyConnector`)
- **Repository Root (server path)**: Absolute path to your local `oe_csp_sdk` repo root
- **Connector Objects**: Objects your connector supports (tables, views, etc.)
- **Connector Icon**: Icon file shown in the UI
- **Connector Capabilities**:
    - **Protocol**: Select the connector protocol (`REST`, `SOAP`, `SDK`, `JDBC`, or `Other`)
    - **Core Capabilities**: Enable what the connector should expose (for example, Crawling, Query Sheet)
    - **Crawler Options / Crawl Types / Crawler Preference**: Select only what your connector supports
- **Research References**: Optional links/notes for implementation context

Choose how you want the generated connector module delivered:

- **Download**: Generates a zip (named after your connector) and downloads it to your machine.
    1. Extract the zip.
    2. Move the generated connector folder into the `oe_csp_sdk` repository root (or wherever you do connector development).

- **Publish**: Generates the connector module **directly into** the working repository you provided in **Repository Root (server path)** (no zip download).

Next steps depend on how you generated the connector:
- **If you used Download**: Open the generated `INSTRUCTIONS.txt` and follow the listed integration steps (parent `pom.xml`, `csp-api`, and `assembly` updates).
- **If you used Publish**: The generator writes the connector module and applies the wiring/dependency updates automatically (no manual changes required).

This gives you a working scaffold so you can focus on implementing connector logic (connection, metadata, and query behavior).

## B. Generate with Maven Archetype (CLI)

Scaffold a new connector from the command line using the Maven archetype. This generates all required classes with empty method stubs ready for implementation.

**Step 1 — Build and install the archetype once from the repo root:**

```bash
mvn clean install -pl connector-archetype -am
```

**Step 2 — Generate the connector** (non-interactive mode recommended — all parameters supplied up front):

```bash
mvn archetype:generate \
  -DarchetypeGroupId=com.ovaledge \
  -DarchetypeArtifactId=oe-csp-connector-archetype \
  -DgroupId=com.ovaledge \
  -DarchetypeVersion=<version> \
  -DartifactId=<artifactId> \
  -Dversion=<version> \
  -DclassPrefix=<classPrefix> \
  -DsdkVersion=<version> \
  -DinteractiveMode=false
```

**Step 3 — Maven Arguments:**

| Argument | Purpose |
|---|---|
| `-DarchetypeGroupId=com.ovaledge` | Fixed — do not change. |
| `-DarchetypeArtifactId=oe-csp-connector-archetype` | Fixed — identifies the connector archetype. Do not change. |
| `-DarchetypeVersion=<version>` | Must match the `oe_csp_sdk` parent version (e.g., `1.0.0-SNAPSHOT`). |
| `-DgroupId=com.ovaledge` | Maven groupId for the generated connector module. |
| `-DartifactId=<artifactId>` | Module name — one lowercase word, no hyphens (e.g., `amazonq`). Becomes the package segment `com.ovaledge.csp.apps.<artifactId>`. |
| `-Dversion=<version>` | Version of the generated connector — typically matches `oe_csp_sdk` (e.g., `1.0.0-SNAPSHOT`). |
| `-DclassPrefix=<classPrefix>` | PascalCase prefix for generated classes (e.g., `AmazonQ`). Produces `<classPrefix>Connector`, `<classPrefix>MetadataService`, etc. |
| `-DsdkVersion=<version>` | Version of `csp-sdk-core` the connector depends on — use the same as `oe_csp_sdk`. |
| `-DinteractiveMode=false` | Disables Maven prompts. Omit to run interactively. |

**Step 4 — Replace placeholders:**

| Placeholder | Example |
|---|---|
| `<version>` | `1.0.0-SNAPSHOT` |
| `<artifactId>` | `amazonq` |
| `<classPrefix>` | `AmazonQ` |

Full example:

```bash
mvn archetype:generate \
  -DarchetypeGroupId=com.ovaledge \
  -DarchetypeArtifactId=oe-csp-connector-archetype \
  -DgroupId=com.ovaledge \
  -DarchetypeVersion=1.0.0-SNAPSHOT \
  -DartifactId=amazonq \
  -Dversion=1.0.0-SNAPSHOT \
  -DclassPrefix=AmazonQ \
  -DsdkVersion=1.0.0-SNAPSHOT \
  -DinteractiveMode=false
```

**Step 5 — Conventions:**
- **artifactId:** Single lowercase word (e.g., `amazonq`). No hyphens.
- **classPrefix:** PascalCase (e.g., `AmazonQ`).

**Step 6 — Next steps:** Follow the generated `INSTRUCTIONS.txt` in the connector module root to add the module to the parent POM, `csp-api`, and assembly (same steps as in [Connector Assembling](08.Assemble_the_new_connector.md)).

**Step 7 — Icon:** Place `amazonq.png` (or similar) under `amazonq/src/main/resources/icons/` for the connector icon shown in the OvalEdge UI.

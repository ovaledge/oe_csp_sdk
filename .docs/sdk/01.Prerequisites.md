## OvalEdge SDK Developer Registration

Before you can access the SDK libraries or JFrog artifact repository, you must be registered as an OvalEdge SDK Developer. To register, send the following information to **developer@ovaledge.com**:

- **Organization Name / Individual Name**
- **Developer Type** (Partner / Contractor / Service Provider / Open Source Contributor)
- **Primary Contact** — Email Address and Phone Number
- **Intended Connector Use Case**
- **Target Data Source(s)** *(e.g., Salesforce, SAP, Snowflake, API Systems, Analyical Systems, JDBC sources)*

Upon submission, the OvalEdge team will verify your details, issue the applicable SDK agreement for signature, and provision your JFrog credentials.

> **Version Compatibility:** Connector SDK versions are tied to OvalEdge platform release trains. A version compatibility matrix will be published at [https://docs.ovaledge.com/connectors/sdk-compatibility](https://docs.ovaledge.com/connectors/sdk-compatibility) — check this before starting development to ensure you are building against the correct SDK version for your target OvalEdge deployment.

# 7. Compile and Build

From the **repository root:**

| Goal | Command | Result |
|---|---|---|
| Build one connector | `mvn clean install -pl amazonq -am` | Builds `amazonq` and its dependencies. |
| Build everything | `mvn clean install` | Builds all modules (connectors, csp-api, assembly). |
| Build assembly JAR only | `mvn clean install -pl assembly -am` | Builds all modules and produces `assembly/target/csp-sdk-*.jar`. |

---

# 7.1. Run and test

## Option A – Unified API (all connectors)

1. Build: `mvn clean install -pl csp-api -am` (or `mvn clean install`).
2. Run: `java -jar csp-api/target/csp-api-*.jar` — default port is **8800** (see `csp-api/src/main/resources/application.properties`).
3. Open the built-in UI at [http://localhost:8800/](http://localhost:8800/) to select a connector, set connection attributes, and call connection validate, metadata, and query.
4. Call REST endpoints with `serverType` in the request body (e.g., `"serverType": "monetdb"`).
    - `GET /v1/info` does **not** require a request body.
    - Metadata / query / validate endpoints require a request body and (for some endpoints) query parameters.

Example:

```bash
curl -s http://localhost:8800/v1/info
```

> **Note:** If you changed the default port in `application.properties`, update the `curl` URL accordingly.

## Option B – Standalone connector (quick app)

For a connector that has a `quick` package (e.g., `monetdb`):

```bash
mvn -pl monetdb spring-boot:run
```

Or run the JAR of that module (main class is configured in its `pom.xml`).

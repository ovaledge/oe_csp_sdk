# 2. OvalEdge Connectors Public Repository

The OvalEdge Connectors SDK provides a prebuilt sample repository to develop new connectors.

**https://github.com/ovaledge/oe_csp_sdk**

**Fork this repository** into your own GitHub account before beginning development. Do not develop directly against the upstream repository.

> **Staying in sync with upstream:** The OvalEdge SDK evolves over time. To pull in SDK updates to your fork, configure the upstream remote and rebase periodically:
> ```bash
> git remote add upstream https://github.com/ovaledge/oe_csp_sdk.git
> git fetch upstream
> git rebase upstream/dev   # Use upstream/dev to stay aligned with the active development branch
> ```
> Resolve any conflicts before continuing development. Breaking changes are documented in the SDK `CHANGELOG.md`.
> 
> **Branch targeting:** Rebase against `upstream/dev` for day-to-day sync. Only rebase against `upstream/main` if you need a production-stable baseline (e.g., targeting a specific released version).

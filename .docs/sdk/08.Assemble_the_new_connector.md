# 8. Assemble the new connector

After adding a new connector module (e.g., `amazonq`), wire it into the build and discovery. The pom.xml changes mentioned can be skipped if the new connector source is generated from UI and Published. 

**Step 1 — Parent `pom.xml` (repo root)**

In `<modules>`, add:

```xml
<module>amazonq</module>
```

In `<dependencyManagement><dependencies>`, add:

```xml
<dependency>
  <groupId>com.ovaledge</groupId>
  <artifactId>amazonq</artifactId>
  <version>${project.version}</version>
</dependency>
```

**Step 2 — `csp-api/pom.xml`**

In `<dependencies>`, add:

```xml
<dependency>
  <groupId>com.ovaledge</groupId>
  <artifactId>amazonq</artifactId>
</dependency>
```

This allows the unified API to load and route to your connector.

**Step 3 — `assembly/pom.xml`**

Add the same `<dependency>` (without version) so the assembly JAR includes your connector.

**Step 4 — SPI file**

Already done in Step 6: `META-INF/services/com.ovaledge.csp.v3.core.apps.service.AppsConnector` with your connector's FQCN. The assembly uses `ServicesResourceTransformer` to merge all connector SPI files into one.

**Step 5 — Release properties**

The assembly JAR includes a `release-csp-sdk.properties` file (keys: `csp-sdk.release.version`, `csp-sdk.git.commit.full`, `csp-sdk.git.branch`, `csp-sdk.git.buildtime`). Partners building assembly artifacts outside this repo must ensure their build produces an equivalent `release-csp-sdk.properties`.

Then verify:

```bash
mvn clean install -pl amazonq -am
mvn clean install -pl csp-api -am
# or
./build-csp-sdk.sh
```

Confirm the new connector appears in `GET /v1/info` and is included in the assembly JAR.
